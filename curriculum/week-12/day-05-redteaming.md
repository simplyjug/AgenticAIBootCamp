# Day 5: Red teaming & bias

## Learning objectives

- Run a **structured red team** session with scripts + creative prompts.
- Track **bias** slices (gender, dialect) if applicable — document limitations.

## Hands-on

1. Prepare **30** test prompts spanning jailbreak, injection via retrieved text, tool misuse.
2. Score outcomes pass/fail against a rubric.
3. File issues with **severity** and **repro** steps for failures.

## Concepts

- **Safety vs capability:** aggressive refusals hurt UX — tune policy per product.
