# Week 14: Capstone & interview readiness

**Time:** ~16–24 hours · **Prerequisites:** Weeks 1–13 (you choose depth)

## Outcomes

- Produce an **architecture** for an enterprise RAG / chatbot (multi-tenant, RBAC hooks).
- Implement a **vertical slice**: ingest → retrieve → generate → eval **or** load-test story.
- Prepare **demos**, **metrics**, and **interview** talking points.

## Self-service flow

1. **[docs/VIDEO_RESOURCES.md](../../docs/VIDEO_RESOURCES.md#week-14)**.
2. Days **1–4** drive scope; Day **5** is demo + study plan.

## Reading-only path

This week is the capstone and interview prep phase. Reading-only learners should translate earlier technical work into portfolio stories and architecture reasoning.

- Learn how to document a full AI system from problem to metrics.
- Review how to choose scope for a vertical slice and what is realistic.
- Understand how load testing and evaluation support system claims.
- Prepare interview stories around architecture, tradeoffs, and incidents.

Write down your capstone narrative, one demo script, and one interview story.

## Portfolio checklist

- [ ] Architecture diagram + tech choices with **tradeoffs**.
- [ ] Eval numbers or load-test summary table.
- [ ] GitHub repo clean: README, CI green, `.env.example` only.

## Done when

- [ ] **10-minute demo script** rehearsed (problem → architecture → metrics).
- [ ] List of **20 interview stories** (system design + debugging + ethics).

## Portfolio path

This week’s portfolio artifact should turn your learning into a polished capstone:

- One **architecture / scope document** for the chosen capstone.
- One **metrics or load-test summary** that supports the outcome claim.
- One **interview story** that highlights the biggest technical decision or tradeoff.

If you are reading only, write a capstone narrative that connects the problem, solution, and impact.
