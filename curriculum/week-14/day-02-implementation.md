# Day 2: Capstone implementation sprint

## Learning objectives

- Ship a **vertical slice**: ingest → index → query → eval artifact.
- Use **feature flags** for risky changes; keep `main` releasable.

## Hands-on

1. Create `capstone/` package or top-level app wiring your best labs.
2. Add **README** with run instructions (docker compose optional).
3. Run **golden eval** subset from Week 9; capture scores in `capstone/results.md`.

## Scope guardrails

- Prefer **depth** on one path over breadth on five half-built paths.

## Done when

- [ ] `capstone/README.md` with run + test instructions.
- [ ] Screen recording or transcript of demo (optional) linked in README.
