# Day 1: Capstone architecture design

## Learning objectives

- Produce a **C4-style** diagram: context, container, component for your RAG / chatbot.
- Define **NFRs**: p95 latency, RPO/RTO, cost envelope, data residency.

## Hands-on

1. Write **1-page** product brief: user, data sources, non-goals.
2. Draw architecture (Excalidraw, Mermaid, or PowerPoint — export PNG to `capstone/diagrams/`).
3. List **3 risks** and mitigations (scalability, safety, cost).

## Deliverable template

- Ingestion path, vector store, LLM provider, eval pipeline, monitoring — each with owner team (can be “you” for solo).

## Done when

- [ ] Design doc checked in: `capstone/ARCHITECTURE.md`.
- [ ] Stakeholder interview questions if this were enterprise (security, legal).
